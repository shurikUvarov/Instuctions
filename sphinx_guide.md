
# Гайд по установке и настройке Sphinx с темой RTD и поддержкой Markdown

## Шаг 1: Установка Sphinx

Для начала необходимо установить Sphinx. Выполните следующую команду:

```bash
pip install sphinx
```

## Шаг 2: Установка темы RTD (Read The Docs)

Чтобы использовать тему RTD, необходимо установить её через pip:

```bash
pip install sphinx_rtd_theme
```

После установки нужно подключить тему в конфигурационном файле `conf.py`:

```python
html_theme = 'sphinx_rtd_theme'
```

## Шаг 3: Установка плагина для Markdown

Для работы с Markdown необходимо установить дополнительное расширение. Мы рекомендуем использовать `myst-parser`, так как это современный и активно поддерживаемый плагин:

```bash
pip install myst-parser
```

В файле `conf.py` подключите его:

```python
extensions = ['myst_parser']
```

Также убедитесь, что Sphinx будет принимать файлы `.md` наряду с `.rst`:

```python
source_suffix = ['.rst', '.md']
```

## Шаг 4: Примеры синтаксиса Markdown для создания оглавлений и ссылок

Вот пример структуры документации с оглавлением и ссылками на другие страницы:

### Главная страница (`index.md`)

```markdown
# Добро пожаловать в документацию!

Добро пожаловать в мою документацию. Здесь вы найдёте полезные разделы:

- [Руководство](guide.md)
- [Часто задаваемые вопросы](faq.md)

## Оглавление

```{toctree}
:maxdepth: 2
caption: Содержание
- guide.md
- faq.md
```
```

### Страница с Руководством (`guide.md`)

```markdown
# Руководство

В этом разделе содержится руководство.

## Введение

Добро пожаловать в руководство. Этот раздел познакомит вас с основами использования Sphinx.

## Расширенные темы

В этом разделе рассматриваются более сложные темы для использования Sphinx.

![Пример изображения](_static/guide_image.png)
```

### Страница с FAQ (`faq.md`)

```markdown
# Часто задаваемые вопросы

## Как установить Sphinx?

Просто выполните команду:

```bash
pip install sphinx
```

## Как настроить проект?

1. Создайте файл `conf.py`.
2. Запустите команду `sphinx-quickstart`.
```

## Шаг 5: Структура файлов

После настройки структура файлов может выглядеть так:

```
docs/
│
├── source/
│   ├── _static/         # Для картинок и других статичных файлов
│   ├── index.md         # Главная страница
│   ├── guide.md         # Страница с руководством
│   ├── faq.md           # Страница с часто задаваемыми вопросами
│   └── conf.py          # Конфигурация Sphinx
│
├── build/               # Папка для сборки HTML
└── Makefile             # Makefile для сборки
```

## Шаг 6: Сборка документации

После того как все страницы созданы, необходимо собрать документацию:

1. Перейдите в каталог `docs`:
   ```bash
   cd docs
   ```

2. Выполните команду для сборки документации:
   ```bash
   make html
   ```

Документация будет собрана в папке `build/html`.

## Шаг 7: Автоматическая сборка и обновление 

1. Установить sphinx-autobuild
   ```bash
   pip install sphinx-autobuild
   ```
2. Запуск автозборки
```bash
   sphinx-autobuild source/ build/html
   ```
- source/ — директория, содержащая ваши исходные файлы документации (например, .rst, .md).
- build/html — директория для выходных HTML-файлов.
